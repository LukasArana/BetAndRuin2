Register

UMLUseCase
BetAndRuin :: Use Case Model :: Register

Description

Event flow

Main flow

  1. Anonymous user introduces a username, an age, a password, a name, a surname and an email.
  2. System adds new user to the set of known users

Alternative flow

A1. There is already an account with that username. Account cannot be created.

B1. Anonymous user hasn't specified a username, password, age, name, surname or email. Account cannot be created.

C1. Anonymous user isn't old enough. Account cannot be created.

D1. The specified username is already in use. Account cannot be created.

E1. Anonymous user already has an account. Anonymous user is redirected to log in.
