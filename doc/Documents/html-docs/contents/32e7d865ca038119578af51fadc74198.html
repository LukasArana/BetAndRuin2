UMLUseCase
BetAndRuin :: Use Case Model :: Log in

Description

Event flow

Main flow

  1. Anonymous user introduces a username and a password

  2. System show him the correct GUI.

Alternative flow

A1. There isn't a username or the password is incorrect. A message saying credentials were not found is shown

B1. Anonymous user hasn't specified a username or password. A message saying credentials were not valid is shown.

C1. Anonymous user doesn't have an account. Anonimous user is redirected to register.

D1. Anonymous user forgot the password. Anonimous user is redirected to forgot password.

A2. Anonymous user enters the credentials of an Administrator. Anonimous user is redirected to GUI for Administrator.

B2. Anonymous user enters the credentials of a User. Anonimous user is redirected to GUI for User.
