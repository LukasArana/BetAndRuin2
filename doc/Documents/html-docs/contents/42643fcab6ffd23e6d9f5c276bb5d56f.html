BetAndRuin :: Use Case Model :: Set fees

Description

Event flow

Main flow

  1. Admin selects a date
  2. System displays the events of this date
  3. Admin selects an event
  4. Admin selects a question
  5. Admin introduces a fee for the question
  6. System adds new fee to the question of the selected event

Alternative flow

A1. The selected date is before current day, so fee cannot be added

B5. The selected question has already a fee, so it cannot be added
